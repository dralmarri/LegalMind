# AGENTS.md

## تعليمات إلزامية لأي نموذج

أنت تعمل داخل مشروع معرفة قانونية كويتية دائم الحفظ.

### قبل بدء أي عمل

اقرأ بالترتيب:

1. `00_MASTER_SYSTEM_PROMPT.md`
2. `01_SAVE_AND_RECOVERY_PROTOCOL.md`
3. `RESUME_STATE.md`
4. `data/OBJECT_INDEX.md`
5. `verification/VERIFICATION_QUEUE.md`
6. ملف الفرع ذي الصلة.

### قاعدة عدم فقدان العمل

لا تعتمد على سياق المحادثة بوصفه مخزنًا دائمًا. بعد كل دفعة معالجة، يجب كتابة النتائج فعليًا في ملفات المشروع. إذا لم تكن قادرًا على الكتابة إلى الملفات، صرّح بذلك فورًا ولا تدّع أن العمل محفوظ.

### قاعدة المعاملة الذرية

تعالج كل دفعة باعتبارها معاملة واحدة: إعداد النتائج، التحقق من المعرفات، كتابة ملف الفرع، تحديث الفهرس، تحديث طابور التوثيق، تحديث سجل التغييرات، تحديث حالة الاستئناف، ثم إنشاء نسخة احتياطية أو commit.

إذا فشلت خطوة، لا تضع حالة الدفعة `completed`.

### حظر الاختلاق

يحظر اختلاق نص تشريعي أو رقم مادة أو قانون أو طعن أو دائرة أو جلسة أو مصدر أو إجراء أو سند لنموذج.

### الأصل الموضوعي

تصنيف المبدأ يتبع موضوعه وعنوان مجلد محكمة التمييز، لا اسم الدائرة التي نظرت الطعن.

### حالة التحليل الآلي

أي تصنيف أو ربط أو قاعدة جامعة مستنبطة آليًا توسم `machine_pending_human` ولا تتحول إلى `verified` إلا بعد مراجعة بشرية.
