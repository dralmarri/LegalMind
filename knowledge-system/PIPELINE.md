# خط المعالجة المعرفي التنفيذي

## الأمر القياسي

```bash
python knowledge-system/scripts/ingest_legislation.py \
  /path/to/law.docx \
  --law-id KW-51-1984 \
  --law-name "قانون رقم 51 لسنة 1984 في شأن الأحوال الشخصية" \
  --branch personal-status \
  --function substantive
```

## المراحل

1. قراءة DOCX أو TXT أو Markdown.
2. حساب بصمة SHA-256 للمصدر.
3. حفظ نسخة نصية مستخرجة غير قابلة للاستبدال الصامت.
4. اكتشاف المواد وإنشاء كائن مستقل لكل مادة.
5. إنشاء `index.json` للقانون.
6. تطبيق حالة `operationally_accepted` دون الادعاء بالمراجعة البشرية.
7. تشغيل طبقة العلاقات لربط المواد بالمبادئ والقواعد والنماذج.
8. تحديث الفهرس العام وسجل التغييرات وحالة الاستئناف.
9. تنفيذ commit وpush قبل اعتبار الدفعة مكتملة.

## ضوابط السلامة

- النص المستخرج من DOCX ليس بالضرورة النص الرسمي الخالي من أخطاء OCR.
- لا يُعد عدد المواد المتوقع إثباتًا للاكتمال؛ يقارن بالعدد المكتشف ويُنشأ تنبيه عند الفرق.
- لا يُعدل النص المصدر عند التصحيح؛ يسجل التصحيح في طبقة المعرفة.
- العلاقات الزمنية من نوع `supersedes` أو `specializes` لا تُنشأ إلا مع تحديد النطاق.
- كل مبدأ سابق على قانون إجرائي لاحق يخضع لإعادة تقييم زمني، لا للحذف.

## ما يحتاج إلى أتمتة لاحقة

- تحديث `OBJECT_INDEX.md` تلقائيًا.
- تحليل الكتب والأبواب والفصول.
- استخراج الإحالات الداخلية بين المواد.
- كشف التعديلات والإلغاءات آليًا.
- إنشاء اختبارات استرجاع من العلاقات.
- تشغيل GitHub Actions عند إضافة مصدر جديد.
