# LegalMind Retrieval Engine v1.0

## 1. مسار الاسترجاع

```text
User Query
→ Request Type
→ Legal Branch
→ Topic
→ Court Classification Title
→ Micro Issue
→ Temporal Scope
→ Knowledge Object Type
→ Verified Sources
```

## 2. عدم البحث الشامل ابتداءً

إذا أمكن تحديد عنوان تصنيف محكمة التمييز، يحصر البحث داخله أولًا، ثم توسع النتائج بالعلاقات الثانوية فقط عند الحاجة.

## 3. ترتيب النتائج

1. تطابق المسألة الدقيقة.
2. تطابق عنوان تصنيف المحكمة.
3. النص التشريعي الرسمي النافذ.
4. المبدأ القضائي المباشر.
5. الحكم الكامل.
6. القاعدة الجامعة المدعومة.
7. الإجراء أو الدفع.
8. النموذج القضائي.

## 4. نوع الطعن والدائرة

نوع الطعن والدائرة بيانات توثيقية، ولا يحددان التصنيف الموضوعي. المبدأ التجاري قد يسترجع في الأحوال الشخصية إذا كان موضوعه المصنف كذلك.

## 5. التوسع الدلالي المنضبط

يستخدم:

- `secondary_classifications`
- `semantic_tags`
- `legal_concepts`
- `cross_references`

ولا يستخدم التوسع إذا خالف `negative_links` أو `not_for_use`.

## 6. استرجاع متعدد الآباء

الكائن الواحد يجوز أن يظهر في عدة مسارات. مثال مبدأ متاع الزوجية:

- الاختصاص ← اختصاص دائرة الأحوال الشخصية ← متاع الزوجية.
- الحقوق المالية بين الزوجين ← متاع الزوجية.

يبقى المبدأ بمعرف واحد.

## 7. الاستعلامات العامة والخاصة

- السؤال العام عن الاختصاص يبدأ بالقواعد العامة ثم التطبيقات.
- السؤال عن الغرفة الجعفرية يسترجع مبادئ التنظيم الداخلي والتوزيع بين الغرف.
- السؤال عن النسب يسترجع مبادئ النسب وتصحيح الأسماء، ولا يسترجع بقية المجموعة إلا عند الحاجة.
- السؤال عن مسكن الحضانة أو جوازات السفر قد ينتهي إلى اختصاص مدني إذا كان الطلب لا يتطلب تطبيق قانون الأحوال الشخصية.

## 8. التحقق قبل الإجابة

لا يدخل المصدر في الإجابة النهائية إذا كان:

- غير مطابق للمسألة الدقيقة.
- تاريخيًا دون بيان.
- غير موثق في بياناته الأساسية.
- من نموذج أو مذكرة دون سند أصلي.
- معارضًا لمصدر أقوى دون بيان التعارض.
