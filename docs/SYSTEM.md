## 3) خطوط الإدخال

**أ) صندوق الوارد (الأساسي):** ملفا `X.md` (نص المبادئ، تُفصل بنمط `رقم-`) + `X.md.json` (الوصف: branch/topic/subtopic/…) في `/opt/legalmind-ingest/inbox/` — المحرك يلتقطها خلال ~10 ثوانٍ، يزيل التكرار ببصمة SHA-256 للنص المطبَّع، والإدراج آمن للتكرار (`ON CONFLICT (id) DO UPDATE`).

**ب) الرفع الهرمي من المتصفح:** صفحة «رفع مستند هرمي» — DOCX ← معاينة شجرة ← موافقة ← إدخال (المعالج `/opt/LegalMind/tools/kpkg_docx.py` + `/api/docx/*`).

**ج) الإدراج المباشر (تشريعات/أحكام/صيغ):** سكربتات لصق بنمط ثابت: بيانات JSON مضغوطة gzip+base64 + مُدرج بايثون يعمل بالبيئة الثقيلة ويستورد `normalize_text` من المحرك. القالب المرجعي: `/tmp/ingest_tpl.py` على الخادم (يدعم `usable_as_citation`).

**قاعدة الجلسات:** أدخل دفعاتك كلها ثم **فهرسة واحدة** في النهاية.

## 4) الفهرسة والاتساق

PostgreSQL مصدر الحقيقة؛ Qdrant فهرس مشتق يُحذف ويُبنى بالكامل:

```bash
cd /opt/LegalMind && set -a; . deploy/.env; set +a; \
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 \
/opt/LegalMind/.venv/bin/python -m engine.legalmind_engine reindex
```

- `HF_HUB_OFFLINE=1` **ضروري** — يتخطى تعليق HuggingFace (أخطاء 504).
- النجاح = `"consistent": true` والعددان متطابقان (حاليًا 2581 = 2581).
- تلميح: معرف النقطة يتضمن بصمة محتوى (إصلاح تاريخي لتصادم أفقدَ 233 مبدأ — لا تُعِد القص إلى المعرف القديم).

## 6) استوديو الصياغة `/api/draft`

**السير:** الطلب (نوع + وقائع + مذهب) ← **Haiku 4.5** يفكّكه إلى 3–6 محاور بحث ← تضمين كل المحاور دفعة واحدة (`engine/embed_queries_multi.py` — تحميل واحد للنموذج) ← بحث Qdrant لكل محور (تشريعات 6 + مبادئ 6؛ والاستعلام الكامل يجلب أيضًا حكمين ونموذجين) ← دمج بالنقطة الأعلى وقصّ (12 تشريعًا + 12 مبدأً + 2 + 2) ← جلب النصوص من PG ← **Claude Fable 5** (`claude-fable-5`) مع **تراجع خادمي تلقائي إلى Opus 4.8** عند رفض الأمان ← مخرج بثلاثة أقسام: الفحص الإجرائي / سيناريوهات النجاح والفشل ومواطن القوة والضعف / التنفيذ.

**قواعد الموجّه:** استشهاد حصري من السياق المسترجَع؛ لا اختلاق مواد أو طعون؛ إفصاح صريح عند نقص المصادر؛ تمييز المذهبين؛ فحص الإجراءات السابقة الوجوبية (مركز التسوية م9 ق12/2015، لجنة النسب 53/2026، تحكيم الضرر 127–132).

**الإعداد:** `ANTHROPIC_API_KEY` في `/opt/LegalMind/deploy/.env` (يُقرأ عند كل طلب — لا حاجة لإعادة تشغيل عند تغييره؛ سطر واحد فقط!). تبديل النموذج: متغير البيئة `LEGALMIND_DRAFT_MODEL`. الكود بين علامتي `# ==== استوديو الصياغة` و`# ==== نهاية استوديو الصياغة` آخر `app.py` (نسخ احتياطية: `app.py.bak_draft*`). زمن الطلب 2–5 دقائق (مهلة nginx 600s).

## 7) أوامر التشغيل الشائعة

```bash
# حالة الخدمات
systemctl status legalmind-admin legalmind-ingest --no-pager

# إعادة تشغيل الأدمن (بعد أي تعديل على app.py)
systemctl restart legalmind-admin

# عدّ الكائنات
set -a; . /opt/LegalMind/deploy/.env; set +a
psql "$DATABASE_URL" -c "SELECT object_type, count(*) FROM knowledge_objects GROUP BY 1 ORDER BY 2 DESC;"

# بناء ونشر الواجهة بعد تعديل web/app/page.tsx
cd /opt/LegalMind/web && npm run build && rsync -a --delete out/ /var/www/legalmind-v3/ && chown -R www-data:www-data /var/www/legalmind-v3
# ثم Ctrl+Shift+R في المتصفح
```

**تنبيهات طرفية الويب:** فعّل `bind 'set enable-bracketed-paste on'` قبل لصق نصوص طويلة؛ احذر التصاق حروف عربية خفية ببداية الأوامر (لوحة المفاتيح!)؛ للخروج من عارض psql اضغط `q` أو استعمل `-P pager=off`.
