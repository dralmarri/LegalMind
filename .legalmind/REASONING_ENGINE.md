# REASONING_ENGINE — محرك الاستدلال

> **الحالة: ❌ غير موجود.** توجد قواعد استدلال مكتوبة في `knowledge-system/ontology/personal-status.reasoning-rules.yaml` — **ولا يقرأها أي كود**.

## 1. الغرض

تحويل سؤال قانوني إلى **سلسلة استدلال** يمكن تدقيقها خطوة بخطوة، لا إلى إجابة سردية.

## 2. سلسلة الاستدلال الإلزامية

```text
1. تكييف الطلب        → ما نوع النزاع قانونًا؟
2. تحديد الفرع        → أحوال شخصية؟
3. تحديد القانون الحاكم → 51/1984 أم 124/2019 (المذهب)؟
4. تحديد القانون الخاص اللاحق → 53/2026؟ 12/2015؟
5. تحديد النطاق الزمني → متى وقعت الواقعة؟
6. استرجاع النصوص     → المواد النافذة وقتها
7. استرجاع المبادئ    → المبادئ المطابقة للمسألة الدقيقة
8. كشف التعارض        → هل بين المصادر تناقض؟
9. الاستنتاج          → مع ذكر كل سند
10. إعلان الفجوات     → ما لم يُحسم صراحةً
```

## 3. القواعد الحاكمة الست (مستنبطة من الـ 23 مبدأ)

هذه قواعد **مستنبطة** لا اقتباسات حرفية، وكلها بحاجة إلى مراجعة بشرية:

| القاعدة | المضمون |
|---|---|
| `RULE-PS-JURISDICTION-00001` | الاختصاص يدور مع **طبيعة الطلب** لا صفة الأطراف |
| `RULE-PS-JURISDICTION-00002` | توزيع العمل بين الدوائر **تنظيم داخلي** لا يبطل الاختصاص النوعي |
| `RULE-PS-JURISDICTION-00003` | النسب يختص عند **نزاع حقيقي**، لا مجرد تعديل قيد |
| `RULE-PS-JURISDICTION-00004` | الإرث من الأحوال الشخصية **ولو نُظر أمام دائرة تجارية** |
| `RULE-PS-JURISDICTION-00005` | متاع الزوجية عينًا أو نقدًا → أحوال شخصية |
| `RULE-PS-JURISDICTION-00006` | طلبات الحاضنة **ليست تلقائيًا** أحوالًا شخصية |

## 4. قواعد السلامة — غير قابلة للتفاوض

### 4.1 لا استدلال بلا سند
كل خطوة تحمل `object_id` مصدرها. خطوة بلا سند ⇒ **توقف السلسلة**، لا تخمين.

### 4.2 التعارض يُعلَن ولا يُخفى
عند تعارض مصدرين، **يُعرَض التعارض على المستخدم**. النظام لا «يرجّح» بصمت. الترجيح عمل فقهي، لا عمل آلة.

### 4.3 النطاق الزمني قيد صحة
مبدأ `historical_only` **لا يُستند إليه كقانون نافذ**. القانون الخاص اللاحق يُقدَّم في نطاقه.

### 4.4 حاجز المذهب
لا يُخلط السني بالجعفري. عند ثبوت الجعفري ⇒ 124/2019 أولًا. الخلط **بطلان**، لا ضعف.

### 4.5 التصنيف يتبع الموضوع لا الدائرة
`JUR-PS-JURISDICTION-00020` طعن **تجاري** ويحكم **الإرث** ⇒ يُسترجع في الأحوال الشخصية. نوع الطعن **بيان توثيقي لا تصنيفي**.

## 5. المخرج الإلزامي — أثر الاستدلال

```json
{
  "question": "هل تختص محكمة الأحوال الشخصية بطلب تسليم جوازات الأولاد؟",
  "chain": [
    {"step": "تكييف", "result": "طلب حاضنة", "authorities": []},
    {"step": "قاعدة", "result": "طلبات الحاضنة ليست تلقائيًا أحوالًا شخصية",
     "authorities": ["RULE-PS-JURISDICTION-00006"]},
    {"step": "تطبيق", "result": "الجوازات → اختصاص مدني",
     "authorities": ["JUR-PS-JURISDICTION-00022"]}
  ],
  "conflicts": [],
  "gaps": ["لم تُرفع نصوص قانون محكمة الأسرة 12/2015 — قد يغيّر النتيجة"],
  "confidence": "grounded_with_gaps"
}
```

**حقل `gaps` إلزامي.** النظام يعلن ما لا يعرفه. الصمت عن الفجوة أخطر من الجهل بها.

## 6. الترتيب

محرك الاستدلال **بعد** محرك الاسترجاع. الاستدلال فوق استرجاع لا يعمل هو تخمين مُقنَّع.
