## 5. ما يعمل فعلًا ✅

PostgreSQL 16 · Qdrant v1.14.1 · `legalmind-admin` · `legalmind-api` · `legalmind-ingest` · **`legalmind-backup`**
**طبقة التطبيع** (6 امتدادات) · **محرك الإدخال** · **بوابة السند** · واجهة عربية RTL.

**خط الإدخال كاملًا — مُثبت من طرف إلى طرف على الخادم الحقيقي:**

```text
مصدر (ملف أو نص ملصق)
  → Canonical Markdown (تطبيع تقني لا يمس المحتوى القانوني)
  → بصمة SHA-256 على المتن المُطبَّع → فحص التكرار قبل أي كتابة
  → استخراج (مواد / مبادئ / حكم / نموذج)
  → PostgreSQL (مصدر الحقيقة)
  → Qdrant (فهرس مشتق، يُعاد بناؤه بأمر واحد)
```

| القدرة | الحالة |
|---|---|
| رفع ملف من الواجهة | ✅ DOCX · PDF نصي · HTML · TXT · MD |
| لصق نص من الواجهة | ✅ `/api/paste-text` — الأصل يُحفظ حرفيًا وغير قابل للتغيير |
| تضمين دلالي حقيقي | ✅ `intfloat/multilingual-e5-base`، 768 بُعدًا، Cosine |
| منع التكرار | ✅ SHA-256 + فهرس فريد في القاعدة، idempotent |
| رفض PDF الممسوح | ✅ رسالة عربية مفهومة، بلا سجل فارغ |
| متابعة حالة المعالجة | ✅ `/api/file-status/{id}` تستقصيه الواجهة |
| إعادة بناء Qdrant | ✅ `legalmind_engine.py reindex` |
| حماية القاعدة المستنبطة | ✅ قيدا قاعدة بيانات، لا عُرف تطبيقي |

**التضمين المحظور:** `hash_embedding` **ممنوع نهائيًا**. لم يكن نموذجًا بل hashing trick على SHA-256، فكانت «الحضانة» و«رعاية المحضون» شبه متعامدتين. إعداد النموذج يُقرأ من `engine/embedding.py` **وحده**.
