# CHANGE_POLICY — سياسة التغيير

## 1. المرتبة الأولى — لا يُمس إطلاقًا 🔴

| العنصر | السبب |
|---|---|
| `original_text` بعد الإدخال | المصدر الأصلي لا يُغيَّر |
| معرّفات الكائنات (`knowledge_objects.id`) | يُستشهد بها في مستندات قانونية |
| بوابة التغطية في `case_coverage` | جوهر «لا صياغة بلا سند» |
| مبدأ رفض الصياغة عند نقص السند | الغرض من المشروع |
| حاجز المذهب (سني/جعفري) | صحة قانونية |
| قيد النطاق الزمني | `historical_only` ليس قانونًا نافذًا |
| ملفات الهجرة المُطبَّقة (`001`, `002`) | تُضاف `003` ولا تُعدَّل السابقة |
| حاويات `n8n` و`qcases-postgres` | **مشاريع أخرى على نفس الخادم** |
| المنفذ 55432 | 5432 محجوز لـ `qcases-postgres` |
| `point_id` من `sha256(object_id)` | تغييره يفصل المتجهات عن كائناتها |

## 2. المرتبة الثانية — تغيير بموافقة معمارية

| العنصر | لماذا يحتاج موافقة |
|---|---|
| مخطط قاعدة البيانات | يمس نزاهة المعرفة |
| `hash_embedding` → نموذج حقيقي | **يستلزم collection جديدة وإعادة فهرسة كاملة** |
| `VECTOR_SIZE` / أبعاد Qdrant | لا يمكن خلط متجهات نموذجين |
| الأنطولوجيا (`ONTOLOGY.md`) | العقد المعرفي الحاكم |
| منطق التصنيف في `classify()` | يمس تصنيف كل كائن |
| اشتراطات بوابة التغطية | **تشديدها مسموح، تخفيفها ممنوع** |
| المصادقة | سطح أمني |

## 3. المرتبة الثالثة — تغيير حر

الواجهة (تخطيط، ألوان) · رسائل الخطأ · التوثيق · السكربتات · الفهارس · التسجيل (logging) · الاختبارات.

## 4. قاعدة الاتجاه الواحد

بوابة التغطية **تُشدَّد ولا تُخفَّف**. أي تغيير يجعل الصياغة أسهل بسند أقل هو **نقض للمشروع**، لا تحسين له.

## 5. حذف البيانات

**ممنوع حذف كائنات معرفية.** الإبطال بتغيير `verification_status`.

**استثناء واحد مُصرَّح به:** البيانات التجريبية (`branch = اختبار`، `source_key = TEST-LEGALMIND-ENGINE`). تُحذف لأنها **ليست معرفة قانونية**، ووجودها يلوّث الإحصاءات والاسترجاع. راجع `TASKS/0001.md` مهمة T-05.

## 6. الكود الميت — مُصرَّح بحذفه

`index.html` · `script.js` · `style.css` · `server.js` · `package.json` (الجذر) — تطبيق ثابت قديم على المنفذ 5000، **لا تشير إليه أي خدمة**.

⚠️ **لا يُحذف في هذه المهمة** (المهمة توثيقية بحتة). يُحذف في commit منفصل مخصص، بعد تأكيد صاحب المشروع.

## 7. قاعدة الحقيقة قبل البناء

**لا يُبنى فوق رقم غير مُتحقَّق منه.**

الدرس المستفاد حيًّا: `RESUME_STATE` أعلن **869 مادة تشريعية**. الواقع **صفر** — لا ملف واحد. لو بُني محرك صياغة على هذا الافتراض، لأنتج مذكرات تستشهد بمواد **غير موجودة**، وتُقدَّم إلى محكمة.

**كل رقم في وثائق هذا المشروع يجب أن يكون مُتحقَّقًا منه من قاعدة البيانات أو نظام الملفات — لا من محادثة سابقة ولا من تقرير سابق.**

## 8. قاعدة الترتيب

```text
الصدق  →  المعرفة  →  الاسترجاع  →  الاستدلال  →  الصياغة
```

لا يُبدأ في مرحلة قبل إغلاق ما قبلها. **الصياغة فوق معرفة فارغة تنتج مستندًا قانونيًا كاذبًا** — وهو الخطر الوحيد غير المقبول في هذا المشروع.

## 9. قاعدة الوكيل

الوكيل (Claude Code) **مهندس تنفيذ، لا مهندس معماري**. لا يخترع مواصفة، ولا يوسّع نطاقًا، ولا يبني ميزة لم تُطلب. عند الغموض: **يسأل، لا يفترض.**

عند اكتشاف تعارض بين وثيقة والواقع: **يُبلغ فورًا ولا يصلح الوثيقة بصمت.**
