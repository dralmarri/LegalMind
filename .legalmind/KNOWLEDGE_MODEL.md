# KNOWLEDGE_MODEL — مصدر الحقيقة

## 1. المشكلة التي يحلها هذا الملف

يوجد اليوم **مخزنان للمعرفة لا يعرف أحدهما الآخر**:

| المخزن | المحتوى الفعلي |
|---|---|
| `knowledge-system/**/*.md` | 23 مبدأ + 6 قواعد جامعة، مُصنَّفة ومُراجَعة |
| PostgreSQL + Qdrant | **2 كائن تجريبي** (`branch = اختبار`) |

النظام يقرأ من قاعدة البيانات. إذن النظام **لا يرى المعرفة القانونية إطلاقًا**.

## 2. القرار المعماري الحاكم

```text
knowledge-system/  ──(حقن)──►  PostgreSQL  ──(تضمين)──►  Qdrant
   طبقة المصدر                مصدر الحقيقة              طبقة الاسترجاع
   والمواصفة                   للتشغيل
   (يقرأها الإنسان)          (يقرأها النظام)
```

- **`knowledge-system/` = طبقة المصدر والمواصفة.** يكتبها الإنسان، ويراجعها الفقيه. ليست قاعدة بيانات.
- **PostgreSQL = مصدر الحقيقة الوحيد للتشغيل.** كل ما لا يوجد فيه، **غير موجود** بالنسبة للنظام.
- **Qdrant = فهرس مشتق.** قابل لإعادة البناء بالكامل من PostgreSQL في أي لحظة. لا يحمل حقيقة أصلية.

**القاعدة:** إذا تعارض Markdown مع قاعدة البيانات → قاعدة البيانات هي الحاكمة تشغيليًا، و**التعارض نفسه عيب يجب إغلاقه فورًا**، لا التعايش معه.

## 3. الاتجاه أحادي — لا مزامنة عكسية

الحقن يسير في اتجاه واحد: `Markdown → PostgreSQL`.
**ممنوع** توليد Markdown من قاعدة البيانات. المصدر البشري المُراجَع لا تكتبه آلة.

## 4. تسلسل المعرفة

```text
Source File (مصدر أصلي غير قابل للتغيير — sha256)
   └── Knowledge Object (مادة / مبدأ / حكم)
         ├── original_text    ← لا يُمس أبدًا
         ├── normalized_text  ← تصحيح OCR والتحرير
         └── Relationship ──► Knowledge Object آخر
               └── Synthesized Rule (قاعدة جامعة، تحمل مبادئها الداعمة)
```

## 5. عقد الهوية (Identity Contract)

المعرّفات **دائمة ولا تُعاد تسميتها أبدًا** — لأنها تُستشهد داخل مستندات قانونية.

| النمط | مثال |
|---|---|
| `LEG-{LAW_ID}-ART-{N}` | `LEG-KW-51-1984-ART-338` |
| `JUR-{BRANCH}-{TOPIC}-{NNNNN}` | `JUR-PS-JURISDICTION-00001` |
| `RULE-{BRANCH}-{TOPIC}-{NNNNN}` | `RULE-PS-JURISDICTION-00001` |
| `BATCH-{YYYYMMDD}-{HHMMSS}-{HASH8}` | `BATCH-20260713-225811-3E379D93` |

**تحذير:** المحرك يولّد المعرّفات من `slug()` على اسم الملف/الموضوع. المعرّفات المُختارة يدويًا في Markdown (`JUR-PS-JURISDICTION-00001`) **لن يولّدها المحرك تلقائيًا**. موائم الحقن **يجب** أن يمرر `id_prefix` صراحة في الـ sidecar للحفاظ على الهوية. إهمال هذا يكسر كل الاستشهادات.

## 6. ما هو مفقود في النموذج المُنفَّذ

| العنصر | الحالة |
|---|---|
| القواعد الجامعة (`synthesized_rule`) | ❌ لا `object_type` لها في المحرك |
| العلاقات | ❌ جدول `relationships` = **0 صف** |
| طابور التوثيق | ❌ في Markdown، وجدول `verification_issues` = **0 صف** |
| النطاق الزمني | ⚠️ العمود `temporal_scope` موجود، ولا يملؤه أحد |

## 7. قاعدة عدم الاختلاق

المحرك **لا يخترع** وقائع أو تسبيبًا للمبادئ المختصرة. المبدأ المختصر يبقى مختصرًا. لا يُستنتج حكم كامل من مبدأ. هذه قاعدة سلامة قانونية، لا تفضيل هندسي.
