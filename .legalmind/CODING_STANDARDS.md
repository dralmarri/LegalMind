# CODING_STANDARDS — معايير الكود

## 1. القاعدة الأولى — اتبع الموجود

الكود القائم له أسلوب واضح ومتسق. **طابقه.** لا تُدخل أسلوبًا جديدًا لأنك تفضّله.

## 2. Python (`engine/`, `admin/`)

| القاعدة | الأساس |
|---|---|
| `from __future__ import annotations` في رأس كل ملف | متبع في الملفات الثلاثة |
| تلميحات نوعية حديثة: `list[dict]`, `str \| None` | متبع |
| المكتبة القياسية أولًا | `engine/` يعتمد **psycopg فقط** — لا تُضف تبعية بلا سبب قاهر |
| `psycopg` v3 (لا psycopg2) | `psycopg[binary]==3.2.9` |
| `Jsonb()` لأعمدة JSONB | متبع في المحرك |
| `dict_row` في `cases_api` | متبع |
| استعلامات مُعامَلة (`%s`) دائمًا | **لا f-string في SQL أبدًا** |
| الإصدارات مثبتة بدقة (`==`) | متبع في كل `requirements.txt` |

### 2.1 استثناء واحد مقبول للـ f-string في SQL

`cases_api.py::update_case` يبني `SET` ديناميكيًا:
```python
assignments = ",".join(f"{key}=%s" for key in data)
```
**آمن فقط** لأن `data` مُرشَّحة مسبقًا بقائمة `allowed` بيضاء. **إن أضفت حقلًا، أضفه إلى `allowed`.** إهمال ذلك = ثغرة حقن.

### 2.2 رسائل الخطأ بالعربية

```python
raise HTTPException(404, "القضية غير موجودة")
```
متبع في كل المشروع. المستخدم عربي. **حافظ عليه.**

## 4. Bash (`deploy/`)

```bash
set -euo pipefail
```
إلزامي في كل سكربت. السكربتات **idempotent** (تصلح للتثبيت والتحديث معًا).
**كل سكربت جديد يجب أن يُنشأ بصلاحية تنفيذ** (`chmod +x`) — إهمال هذا هو سبب فشل `legalmind-backup.service` اليوم.

## 6. التعليقات

الكود القائم **شبه خالٍ من التعليقات** — والأسماء تشرح نفسها. لا تُضف تعليقات تصف ما يفعله السطر. علّق **فقط** على قيد قانوني لا يستطيع الكود إظهاره:

```python
# المبدأ 4 مقيد بـ«قبل العمل بالقانون المدني الجديد» — لا يُستند إليه كقانون نافذ
```

## 7. الأسرار

لا تُكتب في الكود إطلاقًا. `deploy/.env` و`deploy/admin.env` **غير متتبَّعين في Git** ولهما `.example`. **لا تلتزم (commit) بملف `.env` أبدًا.**

⚠️ الافتراضيات المُدمجة خطيرة:
```python
DATABASE_URL = os.getenv("DATABASE_URL", "postgresql://legalmind:legalmind@...")
ADMIN_PASSWORD = os.getenv("LEGALMIND_ADMIN_PASSWORD", "change-me")
```
مقبولة للتطوير المحلي، **ويجب ألّا تعمل في الإنتاج**. أضف فحص إقلاع يرفض `change-me`.

## 8. معالجة الأخطاء

الأسلوب القائم: التقط، ثم ارفع `HTTPException` برسالة عربية.
```python
except Exception as exc:
    raise HTTPException(503, f"تعذر الاتصال بقاعدة البيانات: {exc}") from exc
```
`from exc` **إلزامي** — لا تبتلع السلسلة الأصلية.

## 9. قاعدة السلامة القانونية

أي كود يمس تصنيفًا أو سندًا أو حالة توثيق **يجب أن يفشل بصوت عالٍ** لا أن يخمّن بصمت. الافتراضي الصامت في نظام قانوني هو خطأ صامت.

**المثال الحي في الكود اليوم:**
```python
branch = metadata.get("branch", "أحوال شخصية")   # ⚠️ افتراضي صامت خطير
```
ملف بلا `branch` يُصنَّف **أحوالًا شخصية** بصمت. هذا بالضبط ما تحظره هذه القاعدة. يجب أن يفشل بدل أن يفترض.
